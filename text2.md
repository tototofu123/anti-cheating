Preventing cheating in an online environment is a game of "cat and mouse." No single method is 100% foolproof, so the most effective strategy is a layered approach that combines technical restrictions with smart assessment design.

1. Technical Restrictions
These tools create a digital "fence" around the student during the exam.

Lockdown Browsers: Prevent users from opening new tabs, using search engines, or accessing other applications (like Discord or ChatGPT) while the test is active.

IP Monitoring & Device Linking: Ensure only one login is allowed per student and flag if two students are accessing the test from the same IP address.

Proctoring Software: Tools like Proctorio or ProctorU use AI or live human monitors to watch the student’s webcam, track eye movements, and record their screen.

2. Assessment Design (The "Cheat-Proof" Method)
Technology can often be bypassed (e.g., using a second phone), so the way you write the test is your strongest defense.

Question Randomization: Use a "Question Bank" so that every student receives a different set of questions in a different order. Even if they are sitting next to each other, their tests won't match.

Time Constraints: Set a tight (but fair) time limit. If a student has only 60 seconds per question, they won't have enough time to look up answers or text a friend.

One-at-a-Time Display: Show only one question at a time and prevent "backtracking." This makes it harder to take screenshots of the whole test to share with others later.

Higher-Order Thinking: Move away from "What is X?" (which is easily Googled) toward "Apply X to this specific scenario." Open-ended, analytical questions are much harder to cheat on than multiple-choice.

3. Deterrence and Culture
Integrity Pledges: Require students to type out and sign an honor code statement before the first question appears. Psychologically, this "prime" reduces the likelihood of dishonest behavior.

Plagiarism Checkers: For essay-based tests, use tools like Turnitin or specialized AI-detection software to ensure the work isn't copied or generated by a bot.
